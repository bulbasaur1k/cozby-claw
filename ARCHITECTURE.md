# Архитектура приложения `cozby-claw`

## Обзор
`cozby-claw` — это приложение, построенное на Rust, которое объединяет в себе графический пользовательский интерфейс (GUI), логику бизнес-процессов, работу с внешними API, управление агентами и инструментами.

Репозиторий структурирован по принципам Workspaces и содержит несколько крейтов (crates):

| Крат | Путь | Предназначение |
| :--- | :--- | :--- |
| `cli` | `rust/crates/cli` | (Пока не реализован, отсутствует `main.rs`) |
| `gui` | `rust/crates/gui` | Графический интерфейс на базе `egui` |
| `tools` | `rust/crates/tools` | Набор инструментов для агентов |
| `runtime` | `rust/crates/runtime` | Среда выполнения для логики агентов |
| `api` | `rust/crates/api` | Работа с API моделей |

---

## Роль каждого крейта

### 1. `gui` (Graphical User Interface)
Графический интерфейс, реализованный с использованием `egui`. Является основным клиентом для взаимодействия с пользователем.

- **Технология:** `egui` + `eframe`.
- **Основные файлы:**
  - `src/main.rs` — Точка входа, инициализирует и запускает `eframe`-приложение.
  - `src/app.rs` — Основной функционал UI: состояние (сессии, настройки), отрисовка транскрипта, панели инструментов, модальные окна.
- **Взаимодействие:**
  - Создает и управляет фоновым потоком агента.
  - Обменивается сообщениями с потоком агента (`protocol::AgentToUi` и `protocol::UiToAgent`).

---

### 2. `tools`
Центральный репозиторий инструментов, которые могут использоваться агентами (включая UI).

- **Основная функция:** Имплементация 40+ встроенных инструментов (tools) как функций Rust.
- **Архитектура:** Инструменты реализованы как отдельные функции, которые парсят входные данные (`serde_json::Value`) и возвращают JSON или ошибку.
- **Ключевые инструменты:**
  - `bash`, `read_file`, `write_file`, `edit_file`, `glob_search`, `grep_search` — файловые и shell-операции.
  - `WebFetch`, `WebSearch` — веб-поиск.
  - `ToolSearch` — внутренний инструмент для поиска доступных инструментов.
  - `Agent` — для запуска под-агентов.
  - `todo_write`, `Config`, `StructuredOutput` и другие.
- **Создание инструмента:** Вызовется по имени, парсит входные данные, запускает соответствующую функцию и возвращает JSON-результат.

---

### 3. `runtime`
Специализированный среда выполнения для управляющей логики "разумных" агентов (agent workflows).

- **Предназначение:** Управление состоянием сессии (`Session`), маршрутизация инструментов, обработка событий.
- **Состояние:**
  - `Session` — история всех сообщений (user/assistant), используется для сохранения прогресса.
  - `ToolExecutor` — абстракция для выполнения инструментов.
  - `PermissionEnforcer` — контроль прав (read-only, workspace-write и т.д.).
- **Движок агента:** `ConversationRuntime` управляет потоком: принимает промпт, вызывает LLM, обрабатывает tool calls, возвращает результат.

---

### 4. `api`
Обертка для работы с внешними API моделей (например, OpenAI-совместимые).

- **Функция:** Абстрагирует сетевой слой.
- **Интеграции:** Позволяет `runtime` взаимодействовать с облачными (GPT, Claude) или локальными (например, Ollama) моделями через общий `ProviderClient` интерфейс.
- **Гибкость:** `runtime` не зависит напрямую от провайдера, а работает с `api`.

---

## Точки входа и основные потоки

1. **Запуск приложения:**
   - Пользователь запускает бинарник из `cli` (пока недоступно) или `cozby-claw-gui`.
2. **Создание GUI:**
   - Запускается `eframe::run_native`.
   - Создается экземпляр `AgentApp`, инициализирующий каналы для связи с агентом.
3. **Запуск фонового агента (`agent.rs`):**
   - В фоновом потоке инициализируется `ConversationRuntime`.
   - Создается `ProviderClient` для работы с LLM.
   - Цикл событий: ожидание `UiToAgent::Prompt`, обработка, отправка событий `AgentToUi`.
4. **Работа с инструментами:**
   - Когда агент вызывает инструмент (напр., `bash`), `runtime` делегирует его выполнение.
   - `runtime` использует разрешающую логику, чтобы определить, какой конкретно инструмент (`tools`) вызвать.
   - Выполняется функция из `tools` и результат возвращается в агент.

---

## Выводы
- **`gui`** — UI-слой, клиент.
- **`tools`** — библиотека инструментов (интеграции).
- **`runtime`** — ядро бизнес-логики для агентов.
- **`api`** — слой для интеграции с LLM.

Приложение построено вокруг идеи **агента, управляющего выполнением инструментов**, и предлагает как GUI, так и потенциально CLI интерфейс для запуска агентов с различными стратегиями.
